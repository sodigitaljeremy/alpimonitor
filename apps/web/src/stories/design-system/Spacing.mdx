# Spacing

AlpiMonitor utilise l'échelle Tailwind par défaut (4px × multiplicateur). Aucun
token d'espacement custom n'a été ajouté — l'inertie des valeurs `1`, `2`, `4`,
`6`, `8` reste suffisante pour rythmer une seule page scrollable.

## Échelle utilisée

Seules les valeurs listées ci-dessous apparaissent dans `apps/web/src`. Les
multiplicateurs absents ne sont pas interdits ; ils n'ont juste pas eu de
cas d'usage à date.

## Rythme vertical des sections

Une section scrollable a une respiration verticale systématisée. Plus le
contenu est dense (tableaux, cards), plus le padding monte.

| Section               | Padding vertical desktop   | Pourquoi                                                              |
| --------------------- | -------------------------- | --------------------------------------------------------------------- |
| OHeroSection          | `py-10 md:py-16` via inner | Il occupe la viewport entière, le padding est absorbé par la hauteur. |
| OMapSection           | `py-16 md:py-24`           | Carte + légende, espace respirable autour du cadre.                   |
| OWhyLindasSection     | `py-20`                    | Texte + snippet code, rythme intermédiaire.                           |
| OResearchZonesSection | `py-24`                    | Trois cards dark + paragraphes — la plus longue section.              |
| OSiteFooter           | `py-14`                    | Fin de page, ne doit pas surcharger.                                  |

## Containers

Tous les containers sont centrés via `mx-auto` et capés en largeur via les
tokens `max-w-*`. Seuls trois ont été activés :

- **`max-w-2xl`** (42rem / 672px) — utilisé pour les paragraphes de prose
  dans `MSectionHeader__subtitle` afin de garder une longueur de ligne
  confortable à la lecture.
- **`max-w-5xl`** (64rem / 1024px) — inner de OHeroSection et
  OWhyLindasSection, sections narratives plutôt compactes.
- **`max-w-6xl`** (72rem / 1152px) — inner de OMapSection, OResearchZonesSection,
  OSiteFooter. C'est le largeur "reference" du produit.

Aucune section n'utilise `max-w-7xl` ou `max-w-full`.

## Breakpoints

Le projet utilise les breakpoints Tailwind standards mais n'active en
pratique que deux d'entre eux :

- **`md` (768px)** — pivot principal. La majorité des layouts passent d'une
  colonne à deux ou trois à partir de `md`.
- **`sm` (640px)** — utilisé marginalement (légende de la carte, quelques
  grilles d'icônes dans Storybook).
- **`lg` / `xl` / `2xl`** — jamais déclenchés à date. Le contenu reste
  lisible au-delà sans nouveau breakpoint.

Une valeur pixel hardcodée apparaît dans `OHydroChart.vue` : `width < 420`
active une densité réduite de ticks horizontaux. Cette borne est posée en
JS plutôt qu'en CSS parce qu'elle pilote un calcul D3, pas une règle de
layout.
